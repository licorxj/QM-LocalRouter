## 安装与启动

### 快速启动（推荐）

```bash
# 1. 初始化数据库
cd scripts
python init_db.py          # 跨平台
# ./init_db.sh             # Linux/macOS
# init_db.bat              # Windows
cd ..

# 2. 启动所有服务
# Linux/macOS
chmod +x scripts/manage.sh
./scripts/manage.sh start

# Windows
scripts\manage.bat start
```

### 手动启动

```bash
# 终端 1 - 后端
cd backend
venv\Scripts\activate                     # Windows
# source venv/bin/activate                # Linux/macOS
uvicorn app.main:app --host 0.0.0.0 --port 12002

# 终端 2 - 前端
cd frontend
npm install
npm run dev
```

打开浏览器访问 **http://localhost:12001**。

## 添加平台

**平台管理** 页面集成了平台、API Key 和模型三大管理功能。

### 手动添加

1. 点击右上角的 **"+"** 按钮
2. 填写：
   - **名称** — 例如 OpenAI、DeepSeek
   - **协议** — openai / gemini / claude / custom
   - **接口地址** — 开启自动补全时只需输入域名（如 `https://api.openai.com`），关闭时输入完整路径
   - **平台主页** — 可选，平台官网地址
   - **描述** — 可选备注
3. 点击 **保存**

### 热门平台一键添加

1. 点击 **"热门平台"** 按钮
2. 浏览或搜索平台（支持模糊搜索名称、协议、描述）
3. 点击平台条目自动填充名称、协议、地址、图标和主页
4. 确认并保存

### 平台图标

- 在添加/编辑对话框中点击 **"搜索图标"**
- 输入关键词搜索（如 "openai"）
- 从 3×3 图片网格中选择
- 支持百度、必应、搜狗图片搜索

### 协议参考

| 协议 | 说明 | 接口地址示例 |
|------|------|-------------|
| **openai** | OpenAI 兼容 API | `https://api.openai.com/v1` |
| **gemini** | Google Gemini API | `https://generativelanguage.googleapis.com/v1beta` |
| **claude** | Anthropic Claude API | `https://api.anthropic.com/v1` |
| **custom** | 自定义兼容端点 | 任意地址 |

## 管理 API Key

选择平台后，右侧面板上半部分显示该平台的 API Key。

### 添加 Key

1. 点击 **"添加 Key"**
2. 粘贴 API Key 值
3. 设置别名（如 "主 Key"、"备用 Key"）
4. 设置权重（默认 1，用于加权负载均衡）
5. 保存

### Key 状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| **有效** | 绿色 | 测试通过，可正常使用 |
| **无效** | 红色 | 测试失败，不可用 |
| **未测试** | 灰色 | 尚未进行测试 |

### 批量操作

- **测试全部** — 一键检测所有 Key 是否可用
- **删除无效** — 批量移除状态为"无效"的 Key

### 复制 Key

点击 Key 右侧的 **复制图标** 可复制该 Key 的**真实值**（解密后的原文）到剪贴板。

### Key 加密

所有 API Key 使用 Fernet 对称加密（AES-128）加密存储。加密密钥自动生成，保存在 `backend/data/.encryption_key`。

## 管理模型

右侧面板下半部分显示该平台的模型列表。

### 手动添加

1. 点击 **"添加模型"**
2. 填写：
   - **模型 ID** — 上游模型名称（如 `gpt-4o`、`claude-3-opus`）
   - **显示名称** — 可选友好名称
   - **模型类型** — 文本 / 图像 / 视频 / TTS / 嵌入
   - **多模态** — 支持图片/音频/视频输入的模型开启此选项
   - **上下文窗口** — 最大 Token 数（预设选项：4K/8K/16K/32K/64K/128K/200K/1M）
   - **温度** — 默认温度（0–2）
   - **输入价格** — 每百万输入 Token 价格
   - **输出价格** — 每百万输出 Token 价格
3. 保存

### 从上游同步

点击 **"同步平台模型"** 从上游 API 获取可用模型。这会打开一个弹窗：
- 显示上游提供商的所有模型
- 自动检测模型参数（类型、上下文窗口、价格等）
- 已添加的模型标记为"已添加"
- 支持选择后批量添加

**详见下一节。**

## 同步平台模型弹窗

同步弹窗提供强大的交互式模型浏览与添加功能。

### 打开弹窗

点击模型区域工具栏中的 **"同步平台模型"** 按钮。

### 弹窗功能

- **模型列表** — 每条模型显示：
  - 勾选框
  - 模型 ID（含复制按钮）
  - 显示名称
  - 模型类型（带彩色标签）
  - 多模态标识
  - 上下文窗口
  - 输入/输出价格
  - 单个 **添加** 按钮
- **已添加** — 已存在于数据库中的模型以半透明显示并标注"已添加"
- **全选** — 一键全选所有可添加的模型
- **反选** — 反转当前选择状态
- **批量添加** — 一键添加所有选中的模型
- **选中计数** — 实时显示当前选中的模型数量

### 添加模型

**逐条添加：**
点击任一行右侧的 **添加** 按钮即可立即添加该模型。

**批量添加：**
1. 使用勾选框选择模型（或使用全选/反选）
2. 点击顶部的 **批量添加 (N)** 按钮
3. 完成后显示"添加成功 X 个，失败 Y 个"

## 创建路由策略

路由策略定义了请求如何转发到上游平台。

### 创建策略

1. 进入 **路由策略** 页面
2. 点击 **"创建策略"**
3. 填写：

**基本信息：**
- **策略名称** — 客户端请求时用作 `model` 字段值
- **负载均衡** — 选择均衡方式
- **描述** — 可选备注

**Key 策略：**
- **Key 选择** — 轮询 / 随机 / 故障转移
- **Key 切换触发** — 无 / RPM 阈值 / 计数阈值

**高级：**
- **超时** — 请求超时秒数（默认 120）
- **重试次数** — 失败重试次数（默认 2）
- **启用** — 策略开关

4. **添加路由规则**：选择平台 + 模型，设置优先级和权重，点击"添加"
5. 点击 **"创建"**

### 负载均衡策略

| 策略 | 说明 |
|------|------|
| **轮询 (Round Robin)** | 按顺序依次使用各规则 |
| **加权 (Weighted)** | 按权重比例分配（如 3:1 = 75%:25%） |
| **随机 (Random)** | 每次随机选择一条规则 |
| **故障转移 (Failover)** | 按优先级使用，失败时自动切换到下一条 |
| **优先级 (Priority)** | 始终使用最高优先级 |

### Key 策略

| 策略 | 说明 |
|------|------|
| **轮询** | 按顺序依次使用各 Key |
| **随机** | 随机选择一个 Key |
| **故障转移** | 使用第一个 Key，失败时切换到下一个 |

### Key 切换触发

| 模式 | 说明 |
|------|------|
| **无** | 不自动切换 |
| **RPM** | 每分钟请求数超过阈值时自动切换 |
| **计数** | 总请求数超过阈值时自动切换 |

### 策略测试

点击策略卡片上的 **"测试"** 按钮即可发送测试请求，验证路由链路是否正常工作。

## 发送请求

### 请求格式

```
POST http://localhost:12002/v1/chat/completions
```

```json
{
  "model": "your-strategy-name",
  "messages": [{"role": "user", "content": "你好！"}],
  "stream": false
}
```

- **model**: 策略名称（不是上游模型名称）
- **messages**: 标准 OpenAI 聊天消息格式
- **stream**: `true` 启用 SSE 流式输出

### curl 示例

```bash
curl http://localhost:12002/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"my-strategy","messages":[{"role":"user","content":"写一首诗"}],"stream":true}'
```

### OpenAI Python SDK

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:12002/v1",
    api_key="任意值",  # 可填任意字符串
)

response = client.chat.completions.create(
    model="my-strategy",  # 策略名称
    messages=[{"role": "user", "content": "你好！"}]
)
```

### 第三方客户端配置

| 客户端 | 配置方式 |
|--------|----------|
| **ChatBox** | 设置 > 自定义 API > Base URL: `http://localhost:12002/v1` |
| **CherryStudio** | 设置 > 添加自定义 API > Base URL: `http://localhost:12002/v1` |
| **LobeChat** | 设置 > 添加自定义模型 > API URL: `http://localhost:12002/v1` |
| **Open WebUI** | `OPENAI_API_BASE_URL=http://localhost:12002/v1` |
| **Cursor** | 设置 > 模型 > Base URL: `http://localhost:12002/v1` |
| **Continue (VS Code)** | 添加模型 + OpenAI 兼容供应商，配置 Base URL |

> **注意**: `api_key` 可填任意值。`model` 字段必须填策略名称。

## 查看请求日志

**请求日志** 页面展示所有代理转发的请求。

| 列 | 说明 |
|----|------|
| **时间** | 请求发生时间 |
| **策略** | 使用的路由策略 |
| **平台** | 实际使用的上游平台 |
| **模型** | 实际上游模型名称 |
| **Key** | 使用的 API Key（别名） |
| **状态码** | HTTP 状态码 |
| **延迟** | 响应时间（毫秒） |
| **Token** | 提示词 / 补全 / 总计 |
| **流式** | 是否使用流式传输 |
| **错误** | 错误信息（如有） |

### 筛选

支持按以下条件筛选日志：
- **策略** — 仅显示特定策略的请求
- **平台** — 筛选指定平台
- **状态码** — 状态码筛选
- **时间范围** — 自定义日期/时间范围

## 系统设置

点击侧边栏齿轮图标进入 **设置** 页面。

### 外观设置
- **主题** — 暗色 / 亮色模式
- **语言** — 中文 / 英文

### 代理配置
- 显示需要配置到客户端的代理地址
- 提供 curl 测试示例

### 输出协议
| 模式 | 说明 |
|------|------|
| **OpenAI 格式** | 默认 — 兼容大多数客户端 |
| **Claude 格式** | Anthropic Messages API 格式 |
| **Gemini 格式** | Gemini GenerateContent 格式 |

### 路由管理
- **默认模型** — 路由管理器的默认模型，留空使用策略路由

### 备份与恢复

**自动备份：**
1. 设置备份文件夹（默认：`backend/data/backups/`）
2. 启用自动备份并设置间隔（小时）
3. 系统在启动时和配置间隔自动备份

**手动备份：**
点击 **"立即备份"** 按钮创建即时备份。

**恢复数据：**
- **从列表恢复**：点击任意备份条目的"恢复"按钮
- **上传文件恢复**：点击"上传恢复"并选择 `.db` 文件

> **警告**：恢复操作会替换所有当前数据。建议先创建备份。

## 常见问题

### Q: 无法连接到 LocalRouter？
- 后端是否运行在端口 12002？
- 前端是否运行在端口 12001？
- 客户端 Base URL 是否设置为 `http://localhost:12002/v1`？

### Q: "Strategy not found" 错误？
- 策略是否已创建并启用？
- `model` 字段是否设置为策略名称（而不是上游模型名称）？

### Q: 请求超时？
- 在策略设置中增加超时时间
- 检查网络连接到上游提供商是否正常
- 检查上游 API Key 余额/配额

### Q: 如何查看使用了哪个 Key？
- 查看 **请求日志** 页面
- 每条日志都显示了使用的策略、平台和 Key

### Q: API Key 安全吗？
- Key 使用 Fernet 对称加密（AES-128）加密存储
- 存储在本地 SQLite 数据库中
- 系统仅本地运行，无公网暴露

### Q: 如何备份我的数据？
- 使用 **设置 > 备份与恢复** 页面操作
- 启用自动备份进行定期备份
- 或手动复制 `backend/data/app.db` 文件

### Q: 可以在生产环境使用吗？
- 可以。建议使用管理脚本（`scripts/manage.sh`）或 Docker 部署
- 配置 nginx 反向代理和 HTTPS 用于公网访问
- 使用进程管理器（systemd、supervisor）确保自动重启
